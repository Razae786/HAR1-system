# Human Activity Recognition using Pose Estimation

## Description

This project is a Streamlit web application that performs human activity recognition from video files. It uses MediaPipe for pose estimation to extract key body landmarks and a pre-trained Decision Tree classifier to predict activities such as "Standing," "Sitting," or "Working." The application also incorporates rule-based logic (based on joint angles and hand movements) to refine these predictions. This system is designed to be simple, efficient, and easy to understand, making it suitable for educational purposes and semester projects.

## How to Install Dependencies

To run this application, you'll need Python and the following libraries. You can install them using pip:

```bash
pip install streamlit opencv-python mediapipe scikit-learn pandas
```

You will also need to have the model files (`activity_model.pkl` and `scaler.pkl`) present in a `model/` directory in the root of the project. These files are generated by the training script.

## How to Run the App

1.  Ensure you have installed all the dependencies listed above.
2.  Make sure the `model/activity_model.pkl` and `model/scaler.pkl` files are present. If not, you might need to run the training script (details in "Model Training" section).
3.  Navigate to the project's root directory in your terminal.
4.  Run the Streamlit application using the following command:

    ```bash
    streamlit run streamlit_app.py
    ```
5.  Open the URL provided by Streamlit (usually `http://localhost:8501`) in your web browser.
6.  Upload a video file to see the activity recognition in action.

## File Structure

```
.
├── model/                    # Directory containing the trained model and scaler
│   ├── activity_model.pkl    # Trained Decision Tree classifier
│   └── scaler.pkl            # Scaler used for feature normalization
├── app.py                    # Python script containing the model training code AND a copy of streamlit_app.py code
├── streamlit_app.py          # The main Streamlit application code for activity recognition
├── train.csv                 # CSV file containing the dataset used for training the model
└── README.md                 # This file
```

-   **`streamlit_app.py`**: This is the core application. It handles video uploads, uses MediaPipe for pose detection, extracts features, predicts activities using the loaded model and rules, and displays the results.
-   **`app.py`**: This script serves two purposes. Firstly, it contains the code to train the Decision Tree model using `train.csv`. After training, it saves `activity_model.pkl` and `scaler.pkl` into the `model/` directory. Secondly, it also contains a string variable that duplicates the code of `streamlit_app.py` (this might have been for convenience during development or for generating the streamlit app file). For running the application, `streamlit_app.py` is the key file. For training, `app.py` is used.
-   **`model/`**: This directory stores the serialized machine learning model (`activity_model.pkl`) and the feature scaler (`scaler.pkl`). These are essential for the application to make predictions.
-   **`train.csv`**: This CSV file contains the raw pose landmark data (132 features) and corresponding activity labels used to train the model. Each row represents a frame/pose, and the 'Activity' column provides the ground truth.

## How it Works

The application processes video input frame by frame:

1.  **Pose Estimation**: [MediaPipe Pose](https://google.github.io/mediapipe/solutions/pose.html) is used to detect 33 human body landmarks from each frame.
2.  **Feature Extraction**:
    *   The 3D coordinates (x, y, z) and visibility of these 33 landmarks are extracted, resulting in 132 features per frame. These are the primary input to the machine learning model.
    *   Specific joint angles (e.g., knee and hip angles) and hand positions are calculated from the landmark coordinates. These are used for the rule-based logic.
3.  **Model Prediction**:
    *   The extracted 132 pose features are scaled using a pre-fitted `StandardScaler`.
    *   The scaled features are then fed into a pre-trained `DecisionTreeClassifier` model, which predicts an initial activity (e.g., "Walking", "Sitting", "Standing").
4.  **Rule-Based Adjustments**: The initial prediction from the model is further refined using a set of heuristic rules:
    *   **Sitting**: If the average knee angle is below a certain threshold (e.g., < 100 degrees), the activity is set to "Sitting."
    *   **Standing**: If the average knee angle is high (e.g., > 160 degrees) and the hip angle is also high (e.g., > 150 degrees), the activity is set to "Standing."
    *   **Working**: If significant hand movement is detected (based on the change in wrist positions between frames) and the person is not sitting, the activity is set to "Working."
5.  **Prediction Smoothing**: To avoid flickering predictions, the system considers the last `N` predictions and chooses the most frequent one as the final output for the current frame.
6.  **Display**: The processed video, with pose landmarks overlaid and the recognized activity displayed, is shown in the Streamlit interface. An activity summary (percentage of time spent on each activity) is also shown after the video processing is complete.

## Model Training

The model was trained using pose features extracted from various activities. The training script is part of `app.py`.

To train the model (or retrain it if you have new data in `train.csv` format):

1.  Ensure `train.csv` is present in the project root. This file should contain 132 columns for pose features and one 'Activity' column for labels.
2.  Run the `app.py` script directly:
    ```bash
    python app.py
    ```
3.  This script will:
    *   Load data from `train.csv`.
    *   Split the data into training and validation sets.
    *   Scale the features using `StandardScaler`.
    *   Train a `DecisionTreeClassifier`.
    *   Print evaluation metrics (accuracy, classification report, confusion matrix).
    *   Save the trained model to `model/activity_model.pkl` and the scaler to `model/scaler.pkl`.

The `streamlit_app.py` then loads these saved files for performing inference on new videos.
